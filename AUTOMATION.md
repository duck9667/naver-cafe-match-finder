# 자동 실행 루틴 (Claude scheduled task)

이 레포의 스크립트를 1시간마다 자동 실행하고, 결과를 Slack으로 정리해서 보내기 위해 실제로 사용한 Claude 스케줄 작업 지침 원문이다. Claude Code의 scheduled task(또는 동일한 방식의 LLM 기반 cron 에이전트)로 그대로 재사용할 수 있다.

## 스케줄

- 주기: `0 * * * *` (1시간마다)
- 대상: 네이버 카페 "모두의 풋살축구"(haha999)의 용산/도곡 지역 풋살 매칭글

## 지침 원문

```
네이버 카페 "모두의 풋살축구"(haha999)에서 용산/도곡 지역 풋살 매칭(팀 대 팀 매치) 게시글 중 새로 올라온 것을 찾아 Slack으로 알려주는 작업이다.

1. Bash로 다음을 실행 (스크립트가 환경변수로 네이버 API 키를 읽으므로 .env를 먼저 로드한다): `cd <레포 경로> && set -a && source .env && set +a && python3 search_futsal.py`
   - 이 스크립트는 네이버 검색 API(cafearticle)로 검색어들을 검색하고, cafename이 대상 카페와 일치하는 글만 골라서, 제목에서 불필요한 단어를 제거하고, 제외 키워드가 들어간 글(매칭 모집 글이 아니거나 이미 끝났거나 용병 모집 글)은 제외하고, 이전에 본 적 없는(=새 글) 항목만 stdout에 출력한다(제목 + 링크 + description 발췌). 이미 본 글은 seen_links.json에 저장돼 있어 자동으로 제외된다.
2. 출력이 "새로운 게시글 없음"이면 아무것도 하지 않고 종료한다. Slack 메시지를 보내지 마라 (이미 알려준 글은 다시 알릴 필요 없음).
3. 출력에 새 글이 있으면, 각 항목의 제목+description을 네가 직접 읽고 의미를 파악해서 아래처럼 한 줄로 정리한다 (날짜 표기가 제각각이므로 정규식이 아니라 네가 직접 해석할 것):
   - 맨 앞에 `[요일-오전/오후]` 형태로 짧은 태그를 붙인다. 요일은 네가 실제 달력 기준으로 계산하고, 오전/오후는 매치 시작 시간 기준으로 판단한다 (예: 17시 시작이면 오후 → `[일-오후]`, 08시 시작이면 오전 → `[토-오전]`). 시간을 못 찾으면 오전/오후 부분은 빼고 `[일]`처럼 요일만 표시한다.
   - 그 다음 날짜는 "OO년 OO월 OO일" 형식 (예: 26년 6월 28일). 연도가 명시 안 되어 있으면 현재 연도 기준으로 추정. 날짜 자체가 추정/모호하면 "(추정)" 표시
   - 시간은 HH:MM-HH:MM 24시간 형식
   - 위치/구장명
   - 인원수 (5:5, 6:6 등 — description의 "몇 대 몇" 항목 참고)
   - 값을 못 찾으면 그 항목만 "파악불가"로 표기
   - 글이 매치 모집이 아니라 실제로는 용병 모집/팀원 모집/회원 모집/구장 양도 등으로 보이면 이 항목 자체를 통째로 제외한다 (스크립트가 제목 기준으로 1차 필터링했지만, description까지 읽고 한번 더 판단할 것. "매치완료", "마감" 등 종료를 뜻하는 표현이 있어도 제외)
   - 기간 필터: 파악한 매치 날짜가 이 작업이 실행되는 당일보다 이전(과거)이면 그 항목은 통째로 제외한다. 날짜를 파악할 수 없는 글은 제외하지 않고 그대로 포함한다 (날짜 항목만 "파악불가"로 표기).
   - 위 과정에서 한 건도 안 남으면 (모두 필터링으로 제외됨) 역시 Slack 메시지를 보내지 않는다.
4. 위 필터링을 거쳐 최종적으로 1건 이상 남으면 Slack MCP 도구(slack_send_message)로 channel_id="<알림 보낼 Slack 채널 ID 또는 user ID>"에 보낸다:
     ```
     **<카페명> - <지역> 매칭 새 글 (N건)**
     • [일-오후] 26년 6월 28일 17:00-19:00 / 용산 더베이스 5구장 / 6:6 - <URL|링크>
     • [토-오후] 26년 7월 4일(추정) 18:00-20:00 / 용산아이파크몰 실내구장 / 파악불가 - <URL|링크>
     ```
   - Slack 도구는 입력 텍스트를 "표준 마크다운"으로 해석해서 Slack 문법으로 자동 변환하므로, 제목 줄을 볼드로 만들려면 별표 2개 `**텍스트**` 를 써야 한다 (별표 1개는 이탤릭으로 변환되어버리니 절대 쓰지 말 것). 제목 줄에는 이모지를 쓰지 않는다. 링크는 URL을 그대로 노출하지 말고 `<URL|링크>` 로 "링크"라는 텍스트에만 건다.
5. 스크립트 실행이나 API 호출에서 에러가 나면 에러 내용만 짧게 Slack으로 보낸다 (반복 실패 시에도 매번 보내도 됨).

추가 컨텍스트: NAVER_CLIENT_ID/SECRET은 레포 루트의 .env 파일에 들어있다.
```

## 주의

- 위 `channel_id`, `카페명`, 경로 등은 실제 사용 환경(Slack workspace, 카페, 로컬 경로)에 맞게 바꿔서 써야 한다.
- 이 지침은 Claude처럼 자연어를 해석하는 에이전트 기반 스케줄러를 전제로 한다. 순수 cron(예: GitHub Actions)만으로 동일하게 재현하려면 위 2번의 날짜/제목 해석 로직을 별도 코드(정규식 또는 LLM API 호출)로 옮겨야 한다.
